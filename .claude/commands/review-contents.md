# /review-contents Task

When this command is used, execute the following content review and proofreading task.

---

## 개요

기술 문서 및 콘텐츠에 대한 비판적 검토와 탈고를 수행하는 태스크입니다.

## 사용법

```
/review-contents                    # 대화형 모드 (범위 선택)
/review-contents --changes          # 현재 세션에서 변경된 파일만 검토
/review-contents --all              # 전체 콘텐츠 검토
/review-contents <file-path>        # 특정 파일 검토
```

---

## 실행 절차

### Step 1: 검토 범위 결정

인자가 없으면 사용자에게 질문:

**선택지:**
1. **변경 사항만** (`--changes`): 현재 세션에서 생성/수정된 파일
2. **전체 콘텐츠** (`--all`): content/ 디렉토리 전체
3. **특정 파일**: 사용자가 지정한 파일/디렉토리

### Step 2: 대상 파일 수집

- `--changes`: `git diff --name-only` 또는 세션 내 수정 파일 추적
- `--all`: `content/**/*.md` 패턴으로 전체 수집
- 특정 경로: 해당 경로의 `.md` 파일 수집

### Step 3: 검토 체크리스트 실행

각 파일에 대해 다음 7가지 기준으로 검토:

---

## 검토 체크리스트

### 1. 기술적 정확성 (Technical Accuracy)

- [ ] 개념 설명이 정확한가?
- [ ] 코드 예시가 실행 가능하고 올바른가?
- [ ] 다이어그램이 설명과 일치하는가?
- [ ] 최신 버전/관행을 반영하는가?

**발견 시 조치:** 오류를 수정하고 수정 사항을 기록

### 2. 일관성 (Consistency)

- [ ] 용어가 문서 전체에서 통일되어 있는가?
- [ ] 명명 규칙이 일관적인가? (예: Port 네이밍)
- [ ] 코드 스타일이 통일되어 있는가?
- [ ] 표/다이어그램 형식이 일관적인가?

**발견 시 조치:** 불일치 항목을 통일하고 명명 규칙 표 추가 고려

### 3. 완전성 (Completeness)

- [ ] 정의되지 않은 메서드/클래스가 있는가?
- [ ] 끊어진 링크(내부/외부)가 있는가?
- [ ] 설명 없이 사용된 개념이 있는가?
- [ ] 예시 코드에 import 문이 누락되었는가?

**발견 시 조치:** 누락된 정의 추가, 링크 수정

### 4. Anti-patterns 확인

- [ ] 흔한 실수 패턴이 언급되어 있는가?
- [ ] "하지 말아야 할 것"이 명시되어 있는가?
- [ ] 함정(pitfall)에 대한 경고가 있는가?

**발견 시 조치:** Anti-patterns 섹션 추가 권장

### 5. 비판적 재검토 (Critical Review)

- [ ] 주장에 근거가 있는가?
- [ ] 과도한 단순화나 오해의 소지가 있는가?
- [ ] 반론이나 대안이 언급되어 있는가?
- [ ] "왜"에 대한 설명이 충분한가?

**발견 시 조치:** 맥락과 이유 보강

### 6. 내용 보강 (Content Enhancement)

- [ ] 부족하거나 피상적인 설명이 있는가?
- [ ] 추가 예시가 필요한 부분이 있는가?
- [ ] 실무 적용 가이드가 충분한가?
- [ ] 비교표나 요약이 도움이 될 부분이 있는가?

**발견 시 조치:** 심화 내용 추가, 실무 예시 보강

### 7. 가독성 및 구조 (Readability & Structure)

- [ ] 논리적 흐름이 자연스러운가?
- [ ] 초보자가 따라가기 쉬운 순서인가?
- [ ] 섹션 간 전환이 매끄러운가?
- [ ] 목차 구조가 학습 흐름에 맞는가?

**발견 시 조치:** 섹션 순서 재배치, 전환 문구 추가

---

## Step 4: 검토 결과 보고

검토 완료 후 다음 형식으로 보고:

```markdown
## 검토 결과 요약

### 검토 범위
- 대상: [변경 사항 / 전체 / 특정 파일]
- 파일 수: N개

### 발견된 문제

| 파일 | 카테고리 | 심각도 | 문제 | 수정 여부 |
|------|----------|--------|------|-----------|
| ... | 기술적 정확성 | 🔴 높음 | ... | ✅ 수정됨 |

### 수정 완료 항목
- [x] 항목 1
- [x] 항목 2

### 추가 권장 사항
- [ ] 권장 1
- [ ] 권장 2
```

---

## Step 5: 수정 적용

- 발견된 문제 중 명확한 오류는 **즉시 수정**
- 구조 변경 등 큰 수정은 **사용자 확인 후 적용**
- 모든 수정 사항을 추적 가능하게 기록

---

## 심각도 기준

| 심각도 | 설명 | 예시 |
|--------|------|------|
| 🔴 **높음** | 기술적 오류, 잘못된 정보 | 개념 오류, 실행 불가 코드 |
| 🟡 **중간** | 일관성 문제, 누락 | 용어 불일치, 끊어진 링크 |
| 🟢 **낮음** | 개선 권장 사항 | 추가 예시, 구조 개선 |

---

## 출력

- **즉시 수정**: 🔴 높음, 🟡 중간 (명확한 경우)
- **사용자 확인 필요**: 구조 변경, 대규모 추가
- **권장 사항**: 🟢 낮음 (선택적 적용)
